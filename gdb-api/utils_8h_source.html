00547 /* Ensure that V is aligned to an N byte boundary (B's assumed to be a
00548    power of 2).  Round up/down when necessary.  Examples of correct
00549    use include:
00550 
00551    addr = align_up (addr, 8); -- VALUE needs 8 byte alignment
00552    write_memory (addr, value, len);
00553    addr += len;
00554 
00555    and:
00556 
00557    sp = align_down (sp - len, 16); -- Keep SP 16 byte aligned
00558    write_memory (sp, value, len);
00559 
00560    Note that uses such as:
00561 
00562    write_memory (addr, value, len);
00563    addr += align_up (len, 8);
00564 
00565    and:
00566 
00567    sp -= align_up (len, 8);
00568    write_memory (sp, value, len);
00569 
00570    are typically not correct as they don't ensure that the address (SP
00571    or ADDR) is correctly aligned (relying on previous alignment to
00572    keep things right).  This is also why the methods are called
00573    "align_..." instead of "round_..." as the latter reads better with
00574    this incorrect coding style.  */
